Overview

Task Router is a skill based Task routing to Agent. A task can be generated from any channel, though as of today only Voice is supported.

CSP can create, manage and view Enterprises. In addition, they can add, manage and view users for each enterprise.

Both CSP Admin and Enterprise Admin can create, manage and view Workspace,Workflow, TaskQueue, Worker, Activity etc. To learn more about managing the Task Router features, please read the Enterprise Console documentation.

This document covers the features and functionalities of the TaskRouter (UI) from a CSP point of view.

To find out more about the Task Router API please check the API documentation.

To sign in the Console please visit https://<your_custom_domain>/taskrouterconsole/

Enterprise

Enterprises are customers of the CSP. CSPs can create, manage, delete Enterprises. This is logical grouping of CSP customers and helps to easily manage billing/invoices for Enterprise.

Enterprise List

Users can view a list of all enterprises that have been added under their organization. Enterprises can be filtered based on the name.

Enterprise Details

Users can view details of a particular Enterprise by clicking any enterprise name in the list.

Users can Delete the enterprise by clicking on the Delete Enterprise button.

Add Enterprise

Whenever CSP adds a new Enterprise, it asks for a mandatory User under that Enterprise. This User gets created under Restcomm as a Sub-Account of CSP Organisation. CSP should provision all the Phone Numbers under this Sub-Account in Restcomm as explained in documentation here. These phone numbers are available to Enterprise to assign to TaskRouter.

Follow the steps below to add an enterprise:

  • Click on the Add Enterprise button in the enterprise list page.

  • Enter the name of the new enterprise.

  • An enterprise admin to be assigned to this enterprise. Enter the email Id of the enterprise admin user you’d want to be the admin of the enterprise. All the numbers will be provisioned under this account.

  • Enter the password for the enterprise admin account. This will be set in Restcomm.

  • Enter a friendly name of the enterprise admin user.

Enterprise name,Mail ID must be unique.

Password should meet below criteria:

At least one Uppercase letter(A,B,C,D……​)
At least one Lowercase letter(a,b,c,d,…​…​.)
At least one number(1,2,3,4,5….)
At least one special character(!,@,#,$,%,&,…​…​.)
Length of the password should be greater than equal to 8.

Once the enterprise is created, click on the Workspace link of a particular enterprise to see all other options mentioned as described in the [Enterprise Console documentation].

Search

All the options are explained in Enterprise Console Docs. There is only one more feature exposed for CSP Admin Console is CSP Admin can select the Enterprise as filter to search each of the entities.
