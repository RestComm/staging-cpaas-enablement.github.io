Introduction

Smart 2FA API lets you send One Time Password (OTP) using SMS, Voice or email.

2FA allows you to add a second layer of security to your Application in addition to the user-id and password. The flexibility of using any channel makes sure your users get OTP even when they experience issues with one of the channels.

To learn more about 2FA, check out the Smart 2FA Overview

This document covers features and functionalities of the 2FA UI Console that leverage the underlying API.

To access 2FA Console visit https://<your_custom_domain>/twofa/ and login with your Restcomm Credentials. The features exposed to the Communication Service Provider (CSP) are similar to the ones available for Enterprises. Please visit the 2FA Console for Enterprises documentation to find out more about it.

White Labeling

Restcomm enables you to go to market with your own 2FA brand in a matter of days. 2FA leverages the Restcomm Whitelabel, hence all the work done by user to whitelabel Restcomm Dashboard is used by 2FA as it is. For further details on Restcomm Dashboard whitelabeling please check the Restcomm Whitelabeling Guide.
