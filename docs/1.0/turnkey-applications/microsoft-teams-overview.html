SMS for Microsoft Teams Overview

Introduction

Telestax Message Exchange for Microsoft Teams helps Communications Providers enable their Business Customers to send and receive SMS using Microsoft Teams and to manage and monitor usage. Business Customers can send two-way SMS from within the application.

To enable Business Customers on Microsoft Teams, below steps need to be followed.

  • Request Access Tokens from the Business Customer Microsoft Teams Admin as explained in the documentation here

  • Create a Business Customer account and set up tokens received from the previous step

  • Create a mapping (via the API or the user interface of the Console) within the Business Customer account

  • Start sending messages

Prerequisite

In order to complete the aforementioned steps, you will need:

  • Microsoft Teams to be set up for your Business Customers.

  • Must have followed the steps mentioned in the document here

  • Working knowledge of cURL and a way to make HTTP API calls (either using cURL or Postman. We recommend Postman if you are not familiar with cURL) Or you can directly use the UI Console exposed by Restcomm.

  • An understanding of the Restcomm API for Microsoft Teams or Console and how to add mappings.

  • An activated Restcomm Cloud account.

You need to reach out to your Technical Account Manager to make sure your account has been correctly set up for Microsoft Teams SMS enablement.

Terminology

As part of this documentation, the following terms will be used to describe specific individuals, concepts, and organizations.

Communications Provider: Communications Provider generally refers to any Telestax Partner providing communications products, applications, capabilities, or services. The term can refer to operators, carriers, providers, suppliers, or any other entity that delivers communications capabilities to enterprise or business customers.

Business Customer: An enterprise or business customer that relies on a Communications Provider to deliver communications products, capabilities, or services to customers or end users.

Setting Up the First Mapping

You must have a valid, working Restcomm account in order to complete this step. If you do not have one, you can contact us.

The purpose of Microsoft Teams mapping is to associate a phone phone number with an existing Microsoft Teams ID. For example, imagine you have a user with the Microsoft Teams ID of mac@myserver.com and the Restcomm number is +1-919-555-1212.

With the proper mapping all inbound SMS messages to the phone number (+1-919-555-1212) will be sent via Message Exchange, the Restcomm service for Microsoft Teams, to the application and Team of mac@myserver.com. Additionally, any time Joe sends an outbound SMS from his Microsoft Teams account, it will be sent from a mapped phone number.

You can set up your mapping in two different ways, either via the Restcomm Message Exchange UI Console for Microsoft Teams or via the API.

Setting up a new mapping via the Message Exchange Console

  • Log in to you Restcomm Message Exchange Console via your Restcomm Cloud account. You will need to link the console URL once it is deployed.

  • Create a new mapping as explained in Section 4.2 Add Mappings in Docs: Message Exchange for Microsoft Teams Console.

  • Before creating a mapping, you need to create an Enterprise. This is a very important step as the Microsoft Teams Bot has one-to-one mapping with the Enterprise.

  • Add Bot Details(for each Enterprise) as explained in section 2.3 Manage Bot details in Docs: Message Exchange for Microsoft Teams Console
