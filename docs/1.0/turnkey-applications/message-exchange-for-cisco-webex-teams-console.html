Overview

CPaaS Cisco Webex Teams helps service providers enable their Enterprise Customers to send and receive SMS using Webex Teams, and to manage and monitor the usage.

For more details, have a look at CPaaS Cisco Webex Teams General Documentation

This document focuses on the usage of the CPaaS Cisco Webex Teams Console’s features.

The Webex Teams Console can be accessed from https://<your_domain>/webexconsole/.

Home

This is the default page the user will see when signing in. Here the user can find:

  • A number of created enterprises and mappings

  • Sent and received messages in the last 24 hours within the organisation

  • 30 days enterprises usage report

  • Mappings and messages of an organisation

Mappings

To navigate to the page, click on the Mappings on the top header.

Users can view a list of all enterprises that have been added under his organisation.Enterprises can be filtered based on the name.

An Enterprise is a service provider’s customer which can help them understand number of mappings created for specific Enterprise over a specific period of time for billing and management purposes.

Each Enterprise has a list of mappings which represents a mapping between SMS enabled number and a Cisco Webex Teams Id.

Adding an Enterprise

  • Click on the Add enterprise button.

  • Enter the name of the new Enterprise and click on the Add Account button

Enterprise name must be unique.

Once the Enterprise is added, Users can click on Mappings again to list all the Enterprises.Click on the newly added Enterprise to manage Mappings as explained in section below.

The Enterprise Details button provides details of the Enterprise and also an option to add a Bot and a Guest Issuer.

  • Click on Add Bot Token and enter the following details:
    Bot name: Enter a name of the bot
    Bot Access Token: This is a mandatory field.
    Guest Issuer ID: This is an optional field.
    Guest Issuer Shared Secret: This is an optional field.

  • Click Done

For more information on how to obtain the above field details, refer to the CPaaS Cisco Webex Teams General Documentation

Enterprise Mapping Details

  • If the user clicks on any enterprise in the list, he will see a paginated list of mappings that belong to the enterprise.

  • Name is the friendly name given to the mapping.Number is the external address (SMS enabled number). webexId is the Webex Teams Id that is mapped to the external number.The last column contains the timestamp (when the mapping was last updated) and information about the user who updated it.

  • Users can edit the name of a particular mapping.

  • Users can delete a particular mapping by clicking on the delete icon that appears by hovering the mouse on a specific row.

Add a Mapping

  • Users can reach add mapping page by clicking on Add Mappings button.

  • The user needs to input below 3 fields:

    • Friendly name for the mapping.

    • External address i.e. the number to be SMS enabled.

    • Webex teams id or multiple ids separated by comma, if it has been set up for group messaging.

  • Users can add single or multiple mappings under a particular enterprise.

  • Phone number has to be unique across the system.

  • While adding, if all the mappings are valid then user will see the mapping list else user will see the rejected mappings where user can modify the webex teams id or number.

  • When a mapping is added, the number is provisioned on CPaaS and also the number is SMS enabled byTelestax

Provisioning Bot and Guest Issuer at the Communications Provider Level

Communications Providers can provision a Bot and a Guest Issuer as follows.

  • Click on the User  Configuration on the top-right corner of the Console as shown on the screenshot below.

  • Once you click on Configuration, the Webex Bot details page appears.

  • Click on the Add Bot Token button.

  • Provide the following details:
    Bot name: Enter a name for the bot
    Bot Access Token: This is a mandatory field.
    Guest Issuer ID: This is an optional field.
    Guest Issuer Shared Secret: This is an optional field.

  • Click Done

For more information on how to obtain the above field details, refer to the CPaaS Cisco Webex Teams General Documentation

  • Click Done. The details you provided will be provisioned globally. You can view them by clicking on the eye icon.

Search

Enterprise Search

The user can perform a search on the enterprises of the organization. Below are the supported filters for enterprise search:

  • Account Scope:

    • Search in Current Account: Shows the list of Enterprises that are created by the logged in user.

    • Search in All Account: Shows the list of all the enterprises under the organisation.

    • Search in Specific Account: To search enterprises that are created by a particular user of the organisation, the user is presented with a list of users of that organisation in a popup window. The user can select the CPaaS account on that popup window.

  • Start Date: Shows the list of enterprises that were created on or after this date.

  • End Date: Shows the list of enterprises that were created on or before this date.

  • Name: Shows the list of enterprises whose name are partially or fully match with the given String:

Mappings Search

The user can perform a search on the enterprises of the organization. Below are the supported filters for enterprise search:

  • Account Scope:

    • Search in Current Account: Shows the list of mappings that are created by the logged in user.

    • Search in All Account: Shows the list of all the mappings under the organisation.

    • Search in Specific Account: To search mappings that are created a particular user of the organisation, the user is presented with a list of users of that organisation in a popup window. The user can select the CPaaS account on that popup window.

  • Start Date: Shows the list of mappings that were created on or after this date.

  • End Date: Shows the list of mappings that were created on or before this date.

  • Enterprise: Shows the list of mappings that belong to this enterprise.

  • Name: Shows the list of mappings whose name partially or fully matches the entered string.

  • Number: Shows the list of mappings whose external address partially or fully matches the entered string.

  • WebexId: Shows the list of mappings whose Webex teams Id partially or fully match the entered string.

Messages Search

The user can search the messages of the organization by applying below filters.

  • Start Date: Shows the list of messages created on or after this date.

  • End Date: Shows the list of messages created on or before this date.

  • Enterprise: Shows the list of messages that belong to this enterprise.

  • To: Shows the list of messages that are sent to this number or Webex Teams Id

  • From: Shows the list of messages that are sent from this number or Webex Teams Id

  • Direction: Shows the list of messages of particular direction (INBOUND, OUTBOUND). Inbound means messages coming from mobile to Webex Teams. Outbound means messages sent from Webex Teams to mobile.

  • Type: Shows the list of messages of particular type (SMS, MMS). Please note MMS is still not supported.

  • Channel Status: Shows the list of messages with this channel status.

  • Channel Error Code: Shows the list of messages with this Channel Error Code.

  • Channel Error Message: Shows the list of messages with this Channel error message.

Usage Records

Usage Records provides usage statistics for Enterprises, Mappings and Messages.

Enterprise Usage Records

The user can view the number of Enterprises created for a given time period. Below filters are supported.

  • Account Scope:

    • Search in Current Account: Shows the number or enterprises created by the logged in user.

    • Search in All Account: Shows the number of enterprises created under the organisation.

    • Search in Specific Account: Searches the number of enterprises created by a particular user of the organisation. The user is presented with a list of users of that organisation in a popup window. The user can select the CPaaS account on that popup window.

  • Start Date: Shows the number of enterprises that were created on or after this date.

  • End Date: Shows the number of enterprises that were created on or before this date.

  • Sub resource:

    • Daily: Shows the number of enterprises created on a daily basis.

    • Monthly: Shows the number of enterprises created on a monthly basis.

    • Yearly: Shows the number of enterprises created on a yearly basis.

    • Today: Shows the number of enterprises created today.

    • Yesterday: Shows the number of enterprises created yesterday.

    • This Month: Shows the number of enterprises created this month.

    • Last Month: Shows the number of enterprises created last month.

Mappings Usage Records

The user can view the number of mappings created. Below filters are supported

  • Account Scope:

    • Search in Current Account: Shows the number of mappings created by the logged in user.

    • Search in All Account: Shows the number of mappings created under the organisation.

    • Search in Specific Account: Searches the number of mappings that are created by a particular user of the organisation. The logged in user is presented with a list of all users of that organisation in a popup window. The user can select the CPaaS account on that popup window.

  • Start Date: Shows the number of mappings created on or after this date.

  • End Date: Shows the number of mappings created on or before this date.

  • Enterprise: Shows the record of mapping that belong to this enterprise.

  • Sub resource:

    • Daily: Shows the record of mappings on a daily basis.

    • Monthly: Shows the record of mappings on a monthly basis.

    • Yearly: Shows the record of mappings on a yearly basis.

    • Today: Shows the record of mappings today.

    • Yesterday: Shows the record of mappings on yesterday.

    • This Month: Shows the record of mappings this month.

    • Last Month: Shows the record of mappings last month.

Messages Usage Records

The user can view the number of message records of the organization by applying the following filters.

  • Start Date: Shows the number of messages created on or after this date.

  • End Date: Shows the number of messages created on or before this date.

  • Enterprise: Shows the number of messages that belong to this enterprise.

  • Direction: Shows the number of messages of a particular direction (INBOUND, OUTBOUND).

  • Type: Shows the number of messages of a particular type (SMS, MMS).

  • To: Shows the number of messages that are sent to this number.

  • From: Shows the number of messages that are sent from this number.

  • Channel Status: Shows the number of messages with this channel status.
