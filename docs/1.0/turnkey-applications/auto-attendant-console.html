Overview

With Telestax Auto Attendant (AA) Communications Providers (CPs) allow Business Customers (also known as Enterprise) to automate the management of incoming calls. Using a voice menu system, AA functions as a virtual receptionist, directing calls to the appropriate individual or department within an organization. For more information, have a look at the General Documentation.

This document contains information for Communications Providers regarding the AA Console user interface. To access the AA Console, navigate to https://yourdomain.com/autoattendantconsole/

To learn more about the API click here.

To begin, the CP must define at least one Enterprise (as explained in Section 2 Enterprise). Then, for each Enterprise, the CP has to define at least one Phone Number (as explained in Section 4 Phone Numbers). While Extensions are not mandatory, the majority of AA systems will have Extensions associated.

To sign in to an AA Console, the CP will use the same credentials that they have for their Restcomm Account.

Only CP Admins under Restcomm can have access to AA. This restriction is because a CP Admin can define a new Enterprise in AA systems. In the process, a Sub Account must also be added. As of today, Restcomm only allows top level Admins to define a Sub Account. A Sub Account cannot define another Sub Account under it.

Enterprise

CPs serve Enterprises or Business Customers. Through their own Accounts, CPs can create, manage, or delete Business Customer Sub Accounts. By setting up Sub Accounts for Business Customers, the CP can easily manage AA and invoices for a specific Business Customer.

Enterprise List

Users can view a list of all Enterprises that have been added under their organization. Enterprises can be filtered based on the name.

The Enterprise list also provides a link to navigate to the AA system configured under that Enterprise.

Add Enterprise

Follow the steps below to add an enterprise:

  • Click on the “Add Enterprise” button in the enterprise list page.

  • Enter the name of the new enterprise.

  • An Admin has to be assigned to this Enterprise. Enter the email ID of the Enterprise Admin user you would like to be the admin of the enterprise. All the numbers will be provisioned under this Restcomm account.

Never share these newly created Business Group Admin credentials with the Enterprise. That is needed because all Numbers added by the CP Admin are registered under this newly created Business Group Admin (Sub Account) under Restcomm. If the Enterprise has access to Restcomm, they can sign in and change the settings. Once an Enterprise is created, please go to the User Management section as explained in Section 6 User Management, and create a new BG Admin. Then share these credentials with the Enterprise.

  • Enter the password for the enterprise admin account. This will be set in Restcomm.

  • A Calling Line Identity Presentation (CLIP) Phone Number is optional. If defined, this phone number will be used for all outbound calls as Extension as well as Transfer to Phone. If not defined, the phone number of the caller will be displayed. This feature can be helpful for a CP in the event their SBC is set up to take calls only from configured numbers.

  • Enter a friendly name for the Enterprise Admin user.

Enterprise name,Mail ID must be unique.

Password should meet the following criteria:
At least one Uppercase letter(A,B,C,D……​)
At least one Lowercase letter(a,b,c,d,…​…​.)
At least one number(1,2,3,4,5….)
At least one special character(!,@,#,$,%,&,…​…​.)
The length of the password should be greater than or equal to 8.

Once an Enterprise is created, click on the AA system link to see all other options mentioned below.

Edit Enterprise

To edit an Enterprise, click on the Enterprise name. The CP can edit the name, Calling Line Identity Presentation (CLIP) Phone Number, and the Ringback Tone.

Ringback Tone: The Announcement SID is assigned to this Enterprise as default Ringback Tone. Whenever a caller is waiting to get connected to a call, an Extension, or an external Phone Number, they will hear this Ringback Tone until the call gets connected.

Business Customers can overwrite this while creating a Menu in the AA System.

AutoAttendant System

AA System List

To navigate to the page, click on the Systems on the top header.

  • CP Admin can view a list of all AA systems that have been added under a particular Enterprise.

  • AA System can be filtered based on the name.

  • CP Admin can deactivate/activate the AA system by clicking on the status of any AA system.

  • CP Admin can delete an AA system by clicking on the delete icon that appears by hovering the mouse on a specific row.

AA System

To create a new AA system click on the Add AA System button, provide a name and click on Save & Continue. The CP Admin may complete a full AA System for Enterprise, but ideally the Enterprise Admin should do it for themselves.

To edit an existing AA system click on the name of the AA system.

Both actions will take you to the basic details page that allows users to edit the following:

  • Basic details

  • Announcement

  • Menu

  • Schedule

  • Special Days

  • Error Handling

Each of the above points are explained in detail under the Enterprise Customer Console Documentation.

Phone Number

The CP must set up one or more lines/phone numbers as Auto Attendant phone numbers. The CP can assign these numbers to an AA System as defined in Auto Attendant System but it is best to let the Enterprise Admin take care of that.

Phone Numbers List

To navigate to the page, click on the phone numbers link located on the top header. There, the CP Admin can do the following:

  • View the list of phone numbers under the enterprise.

  • Assign the phone numbers to the AA system by selecting the AA system from the drop down with AA system names.

  • Delete the phone number by clicking on the delete icon that appears by hovering the mouse on a specific row.

Add Phone Number

To add a phone number under the Enterprise click on the Add Phone Numbers button

  • Enter a phone number.

  • Set a friendly name for the phone number and click done.

These numbers will be provisioned under the main Enterprise Admin Sub Account in Restcomm.

Extensions

The CP may set up one or more Extensions. The Enterprise Admin can only use the Extensions defined by the CP.

Extension List

To navigate to the page, click on the Extensions link located on the top header. There, the CP Admin can do the following:

  • View the list of Extensions that are configured under the enterprise.

  • Edit the extension number and spoken name of the number by clicking on the pencil icon in the respective fields (Spoken Name, Extension).

  • Delete the Extension by clicking on the delete icon that appears by hovering the mouse on a specific row.

Add Extension

To add an extension number under the enterprise click on the Add Extensions button. Ideally as the CP Admin, you would want to only set up the Phone Number that will act as Extension and let the Spoken Name and the Extension Number be set up by the Enterprise Admin.

CP Admins have the option to set a Phone Number for the extension as either a Restcomm Client or Phone/PSTN Number. To learn more about Restcomm Client please visit tbe Restcomm Client documentation .

User Management

Users can have any of the following roles:

  • Communications Provider Admin (also known as CSP_ADMIN) can create Enterprise, Auto Attendant System, Phone Number, Extensions and users with role CSP_ADMIN, BG_ADMIN and AA_ADMIN. Users with roles BG_ADMIN and AA_ADMIN are for Enterprise.

  • Enterprise Admin (also known as BG_ADMIN) can create users with role Enterprise Admin and AA Admin (also known as AA_ADMIN).

  • The only difference between Enterprise Admin and AA Admin is that AA Admin cannot add any new users and cannot make changes to Extensions. The rest is all the same

Users List

To navigate to the page, click on the User Management link located on the top header. There, the CP Admin can:

  • View the list of accounts registered under the Enterprise.

  • Deactivate/activate the account by clicking on the status of any account.

  • Delete the account by clicking on the delete icon that appears by hovering the mouse on a specific row.

All the users created are Sub Accounts under that CP Admin Account in Restcomm.

Add Users

Follow the steps below to add an user:

  • Click on the Add User button in the User management page.

  • Enter the name of the new user.

  • Enter email id of the user.

  • Enter the password for the user account. This will be set in RC.

  • Set a role for the user (BG ADMIN, AA ADMIN).

  • Set the status of the account (ACTIVE,INACTIVE).

Mail ID must be unique. Password should meet below criteria:

  • At least one Uppercase letter(A,B,C,D……​)

  • At least one Lowercase letter(a,b,c,d,…​…​.)

  • At least one number(1,2,3,4,5….)

  • At least one special character(!,@,#,$,%,&,…​…​.)

Length of the password should be greater than equal to 8.

Search

To navigate to the page, click on the Search link located on the top header. The CP Admin can search for Announcements, Menu, and Phone Numbers passing various filters as explained in the Enterprise Console Doc. Additionally the CP Admin can filter based on Enterprise as well.

Usage Records

To navigate to the page, click on the Usage Records link located on the top header. The Usage Records data is based on the Monthly Recurring cost for Phone Numbers (including Extension Numbers ) set up on the AA System. For example, if a number was added on the 5th day of the month and removed on 25th day of the same month, that number will still be counted under the Usage Records data for that particular month. However, from the following month onward, it will not be counted as it was deleted. If the number was not deleted, the Usage Records count will include that number for next month and hence its monthly recurring count. A CP Admin can search for Usage Records for each Enterprise as well.

If you would like to explore the Auto Attendant API, we recommend having a look at the Getting Started with the Auto Attendant API Guide.
