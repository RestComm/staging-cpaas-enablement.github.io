Overview

CPaaS Cisco Webex helps service providers enable their Enterprise customers to send and receive SMS using Webex Teams and to manage and monitor their usage. Enterprise customers can send two-way SMS.

If you would like to start using the CPaaS Cisco Webex Teams feature, you need to complete the following steps:

  • Create Bot and Guest Issuer as described in Creating a Bot and Creating Guest Issuer sections.

  • Provision the Bot and Guest Issuer as described in Provisioning Bot and Guest Issuer section.

  • Create a mapping (via the API or the user interface of the Console)

  • Start sending messages

Prerequisite

In order to complete the aforementioned steps, you will need:

  • An activated CPaaS Cloud account.

  • Webex Teams setup for your Enterprise customers

  • Working knowledge of cURL and a way to make HTTP API calls (either using cURL or Postman; We recommend using Postman, if youre not familiar with cURL.) Or you can directly use the interface of the CPaaS Cisco Webex Console.

  • An understanding of how to add mappings via the CPaaS API for Cisco Webex Teams or the Console.

You need to reach out to your Technical Account Manager to make sure your Account has been correctly set up for Cisco Webex Teams SMS enablement.

Setting up Your First Mapping

You must have a valid, working CPaaS account in order to complete this step.If you do not have one, you can contact us.

The purpose of Webex Teams mapping is to associate a phone phone number with an existing Cisco Webex Teams Id.For example, imagine you have a user with the Webex Teams id of joe@myserver.com and CPaaS number is +1-919-555-1212.

It also supports setting up Group Messaging where the number +1-919-555-1212 is mapped to multiple Webex Teams Ids, like joe@myserver.com, bob@myserver.com, etc.

Mapping will tell CPaaS for Cisco Webex Teams service to send all inbound SMS messages, sent to phone number (+1-919-555-1212), to Webex Teams Space.Where joe@myserver.com is part of, or in case of Group Messaging, both Joe and Bob are added in that space.Additionally, any time Joe (or Bob if part of the same space) sends an outbound SMS from the Cisco Webex Teams account, it will be sent from a mapped phone number.

You can set up your mapping in two different ways, either via the CPaaS Message Exchange for Cisco Webex Teams Console or via the API.

Setting Up a New Mapping via Cisco Webex Teams Console

  • Log into your CPaaS Message Exchange for Cisco Webex Teams Console via your CPaaS cloud account

  • Create a new mapping as explained in Add Mappings Console documentation.

  • Before creating a mapping, you need to create an Enterprise. This is a very important step as Cisco Webex Bot has a one-to-one mapping with the Enterprise.

  • Create Bot and Guest Issuer as explained in Creating a Bot and Creating Guest Issuer sections.

Receiving an SMS/message in Webex Teams

Once youve completed all of the steps above, youre ready to test.

To begin, send a message from a PSTN number to the mapped number. You should see the message appearing in the Webex Teams space. The message should appear in a new space if you are receiving a message for the first time. All subsequent messages will appear in the same space. The default space name is set to Phone Number:xxx, where phone number is the sender’s number and xxx is the name provided while creating the mapping. You can rename the space to something more meaningful, for example, Ryans Mobile Phone:Group 1.

Replying to received SMS/message

To reply to received message follow the steps below:

  • In newly created space, refer to the Bot with @ prefix and then type your message.

If you have Guest Issuer details provisioned at the Enterprise or Communications Provider level, you dont have to use the @ prefix. You can directly type the message.

  • Hit enter. Your message will be sent as SMS to the recipient’s mobile number.

Sending an SMS/message to a New Number from Webex Teams

To send SMS to a new number, follow below steps:

  • Add newly created Bot as Contact in your Cisco Webex if not done earlier.This is a one time process.

  • Send any private message to this Bot

  • It should respond back with Card that allows you to enter the receiving partys phone number

    • If you are part of multiple mappings, it also allows you to select via which mapping you want to send out SMS.

  • Enter the recipient’s phone number and hit Submit.It should create a new space with recipient’s phone number:Mapping name as Space name

  • In newly created space, refer to the Bot with @ prefix and then type your message.

If you have Guest Issuer details provisioned at the Enterprise or CSP level, you dont have to use the @ prefix and you can directly type the message.

  • Hit enter, and your message will be sent as an SMS to the mobile number of the recipient.

Creating a Bot

  • Visit https://developer.webex.com/ then click on Start Building Apps.

  • Login with the Webex for Developers credentials.

  • Click on Create a New App.

  • Click on Create a Bot.

  • Fill the basic details of the bot such as Bot Name, Bot Username, Icon, Description then click Add Bot.

  • Once the Bot is successfully created, you will get Bots Access Token.

Ensure you copy the Access Token for future use.

Creating Guest Issuer

  • Visit https://developer.webex.com/ then click on Start Building Apps.

  • Login with the Webex for Developers credentials.

  • Click on Create a New App.

  • Click on Create Guest Issuer.

  • Enter the Guest Issuer name and click on Add Guest Issuer.

  • Once the Guest Issuer is successfully created, you will get the Guest Issuer ID and Guest Issuer Shared Secret from the Credentials section.

Ensure you copy the Guest Issuer ID and Guest Issuer Shared Secret for future use.

Provisioning Bot and Guest Issuer

To provision the Bot and Guess Issuer you will need to set the following:

  • Bot Access Token: This is a mandatory field.Refer to Creating a Bot

  • Guest Issuer ID and Guest Issuer Shared Secret: These are optional fields.Refer to Creating Guest Issuer
