Android Getting Started Guide

The CPaaS API Android SDK enables you to initiate and receive calls, and listen to call events. This guide will walk you through the process of downloading the CPaaS Android SDK, importing it into your project and testing its capabilities.

Prerequisites

Restcomm Account and Authentication Parameters

  1. To add a Click-To-Call capability to your application, you will need an active Restcomm Account.

  2. After you sign up with Restcomm, you can find your Account SID and Auth Token by navigating to Your profile -→ Account in the Restcomm Console.

If you are not familiar with the Restcomm Console yet, please check out the Restcomm Getting Started Guide

Platform Prerequisites

  1. Support for Android API level 21 or higher

  2. Support for Java 11

CPaaS API SDK Sample Application

You can find our Android SDK sample application in Github. Its sample code demonstrates how to integrate the CPaaSAPI SDK with a kotlin application.

Downloading the CPaaS API SDK

The CPaaS API SDK is available for download in Maven Central.

Maven

implementation ("com.cpaasapi.sdk:1.0.0")

Permissions

Please add the following permissions to the AndroidManifest.xml

    <uses-permission android:name="android.permission.RECORD_AUDIO" />
    <uses-permission android:name="android.permission.MODIFY_AUDIO_SETTINGS" />
    <uses-permission android:name="android.permission.INTERNET" />

Initializing the SDK

To start using the SDK you should first register it by calling the CPaaSAPI.register() method along with the parameters below.

This should be done only once during a session. 
CPaaSAPI.register(
    CPaaSAPISettings(
                customDomain = "api.your-company.com",
                accountSid = "AC288a940234b0d31aaa9df865acc8e492",
                accountToken = "0c4d5959376500dd23d4b3a39126e524",
                appSid = "sid12345",
                clientId = "foobar22",
                BaseURL = "https://yourcompany.com"),

    applicationContext, object: CPaaSAPICb {
        override fun onIncomingCall(call: ICall) {
            runOnUiThread {
                // todo Listening to incoming calls event here
            }
        }

        override fun onRegistrationComplete(success: Boolean) {
            // todo you can start your call here
        }
    }
)

Authorization and Web Socket Connection

If the application registration was successful, the following events will take place:

Authorization – Our servers will authenticate and authorize the use of the SDK
Web socket connection initialization – the SDK will initialize the web socket connection.

The steps above must be successfully completed. Failure in any of the above procedures will prevent the SDK from working properly.

Starting a call

To start a call use the CPaaSAPI.voice.create(…​) and CPaaSAPI.voice.connect(…​) method as follows.

create(): creates and returns a call id which you can use or share with others. connect(): starts the actual connection to the call and returns an API call obj.

    CPaaSAPI.voice.create { createResult ->
        createResult.onSuccess { callId ->
            CPaaSAPI.voice.connect(callId, callOptions) { connectResult ->
                connectResult.onSuccess {  call ->
                    currentCall = call
                    // Here you can:
                    // show a call UI
                    // Start listening to call events: currentCall.eventListener = {...}
                }
            }
        }
    }

The CPaaSCall Object

CPaaSCall is an API object that is returned as a result of the connect() method. The CPaaSCall lets you perform actions on an active call such as muting or ending the call.

call.mute()    (1)
call.endCall() (2)
call.end() (3)
1 Mutes the current call
2 Closes the active call
3 Ends the current call

Answering an Incoming Call

To answer an incoming call, you need to listen to the CPaaSAPICb.onIncomingCall event. When there is an incoming call event, you can use CPaaSAPI.voice.connect(callId, callOptions) to answer it with, using the given callId.

You should not use the create() method in this case. 
override fun onIncomingCall(
    callId: String,
    callerId: String,
    serviceType: ServiceType) {
    CPaaSAPI.voice.connect(callId, callOptions)
}

Listening to Call Events

You can listen to various call events such as: call connected, call ringing, connection failed, call ended, call reconnecting.

call.eventListener = object: CPaaSCallEvents {
    override fun onConnected() {
        // call connected you have audio here
    }

    override fun onRinging() {
        // callee got your call, wait for his action (accept/reject)
    }

    override fun onConnectedFailure(reason: CPaaSReason) {
        // connection failed, check reason
    }

    override fun onCallEnd(reason: CPaaSReason) {
        // call end, you can remove call UI
    }

    override fun onReconnecting(reason: CPaaSReason) {
        // call on reconnecting process, show notification for user
    }
}
