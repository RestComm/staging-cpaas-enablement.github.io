iOS SDK Quick Start Guide

Overview

The CPaaS iOS SDK enabled you to initiate and receive calls and listen to call events. This guide will walk you through the process of getting started with the CPaaS API and iOS SDK.

Getting Started with the iOS SDK

Importing the framework

To import the framework, add the following to your application.

import CPaasAPI

Register the SDK

let settings = CPaaSAPISettings(
            customDomain: "domain",
            accountSid: "sid",
            accountToken: "token",
            appSid: "sid12345",
            clientId: "foobar22",
            PNSToken: "pns55555",
            BaseURL: "https://cpass.dev:3000")

CPaaSAPI.shared.register(settings: settings, cpaasAPICb: self)

extension CLASS_NAME: CPaaSAPICb {

    func onIncomingCall(call: ICall) {
        //TODO: - Listening to incoming calls
    }

    func onRegistrationComplete(success: Bool) {
        //TODO: - You can start your call from here
    }
}
CLASS_NAME stands for the class you will receive the callback to.

Initiating a Call

Now that the iOS SDK is registered, you can start making calls. To initiate a call run the script below.

let call = CPaaSAPI.shared.startCall(
            destinationId: destinationID,
            callOptions: CallOptions(audio: true))

Listening to Call Events

The iOS SDK supports listening to various call events such as: call connected, call ringing, connection failed, call ended, call reconnecting. You can listen to call events as follows.

call.eventListener = self

extension CLASS_NAME: ICallEvents {

    func onRinging() {
        // callee got your call, wait for his action (accept/reject)
    }

    func onConnected() {
        // call connected you have audio here
    }

    func onCallEnd(reason: Reason?) {
        // call end, you can remove call UI
    }

    func onConnectedFailure(reason: Reason) {
        // connection failed, check reason
    }

    func onReconnecting(reason: Reason) {
        // call on reconnecting process, show notification for user
    }
}
CLASS_NAME stands for the class you will receive the callbacks to.

Call Events
Name Description

onConnected

The call is connected and you are receiving audio.

onRinging

The Callee received your call. You have to wait for them to accept or reject the call.

onConnectedFailure

The connection failed. You can check the failure reason.

onCallEnd

The call ended and you can remove the call UI.

onReconnecting

The call is in a process of reconnecting and you can show a notification to the user.
