Android SDK Integration Guide

Overview

The CPaaS SDK enables you to initiate and receive calls, and listen to call events. This guide will walk you through the process of downloading the CPaaS API Android SDK, importing it into your project and testing its capabilities.

Downloading the CPaaS API Android SDK

The CPaaS API Android SDK is available for download in Maven.

implementation ("com.cpaasapi.sdk:1.0.0")

Initialize the SDK

Before you start using the SDK you will need to initialize it first. To do so, you will have to register the application by running the script below.

CPaaSAPI.register(
    CPaaSAPISettings(customDomain = "domain",
                accountSid = "sid",
                accountToken = "token",
                appSid = "sid12345",
                clientId = "foobar22",
                PNSToken = "pns55555",
                BaseURL = "https://mavenir.dev:3000"),
    applicationContext, object: CPaaSAPICb {
        override fun onIncomingCall(call: ICall) {
            runOnUiThread {
                // todo Listening to incoming calls event here
            }
        }

        override fun onRegistrationComplete(success: Boolean) {
            // todo you can start your call here
        }
    }
)

Authorization and Web Socket Connection

If the registration of the application is successful, the following two events will happen.

Authorization - Our servers will authenticate and authorize the use of the SDK.
Web socket connection initialization - We will initialize the web socket connection needed by the SDK.

The previous steps must be successfully completed. Failure in any of the above procedures will prevent the SDK from working properly. 💡 To start using the SDK you will need to invoke the .register() method by running the following script. This can only be done once. A second invocation will take no effect on the SDK. 
CPaaSAPI.register()
CPaaSAPI.register() //The second invoke will have no effect

Starting a call

To initiate a call simply call CPaaSAPI.startCall(…​). <br>

val call = CPaaSAPI.startCall(
    destinationId = destId,
    callOptions = CallOptions(audio = true))

call is an API object that lets you perform actions on the active call such as muting or putting it on hold.

call.mute() (1)
call.hold() (2)
1 Mutes the current call
2 Puts the current call on hold

Answering an Incoming Call

To answer an incoming call, you need to listen for an event on the CPaaSAPICb.onIncomingCall. When there is an incoming call, you can use the call.join() object to answer it.

val currentCall: ICall? = null
override fun onIncomingCall(call: ICall) {
    runOnUiThread {
        currentCall = call
        call.joinCall()
        // Insert your own UI here
    }
}

Listening to Call Events

You can listen to various call events such as: call connected, call ringing, connection failed, call ended, call reconnecting.

call.eventListener = object: ICallEvents {
    override fun onConnected() {
        // call connected you have audio here
    }

    override fun onRinging() {
        // callee got your call, wait for his action (accept/reject)
    }

    override fun onConnectedFailure(reason: Reason) {
        // connection failed, check reason
    }

    override fun onCallEnd(reason: Reason?) {
        // call end, you can remove call UI
    }

    override fun onReconnecting(reason: Reason) {
        // call on reconnecting process, show notification for user
    }
}
