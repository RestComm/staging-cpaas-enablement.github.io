iOS SDK Quick Start Guide

Main Objects

The iOS SDK is using the following main objects:

Property Description

CPaaSAPI

Main API object.

IVoice

Voice Service that allow you to have a call with CPaaS application or receiving a call from a CPaaS application.

CPaaSCall

Perform Actions on an active call

CPaaSAPI

CPaaSAPI is your entry point to the CPaaSAPI SDK.

register

To start using the SDK you should register the SDK by using this function

func register(settings: CPaaSAPISettings, cpaasAPICb: CPaaSAPICb)
Name Description

settings

CPaaSAPISettings object contains parameters needed for registration

cpaasAPICb

CPaaSAPICb is a listener for a general events

logout

This will clear all objects and unregister from receiving further incoming call

func logout()

getFilePath

Call this when you want to get the log file. If file exists you will receive the URL to the path, otherwise you will receive null.

func getFilePath() -> URL?

voice

This protocol of type [IVoice](=ivoice) is your entry point to all voice relevant methods.

var voice: IVoice { get }

<a name="ivoice"></a>IVoice

This protocol is your entry point for handling all voice relevant methods.

create

Creates a callId that can be used to start a call or share it with others.

func create(result: @escaping (CPaaSResult<NSString, CallStartError.NotRegisteredError>) -> Void)
Name Description

result

This parameter is of type CPaaSResult that will return a callId if the request was successful or a CallStartError.NotRegistered if you weren’t registered before.

connect

Call this when you want to start or join a call.

func connect(callId: String, callOptions: CallOptions, result: @escaping (CPaaSResult<CPaaSCall, CallStartError>) -> Void)
Name Description

callId

The callId of the call, this id can be retrieved from create, incomingCall or from sharing

callOptions

This parameter is of type CallOptions to configure some settings when starting the call

result

This parameter is of type CPaaSResult that will return a CPaaSCall if the request was successful or a CallStartError if something went wrong.

reject

Use this when you want to reject an incoming call.

func reject(callId: String)
Name Description

callId

The callId of the call you want to reject

getExistingCall

Returns an existing CPaaSCall by callId or null if the call doesn’t exists.

func getExistingCall(callId: String) -> CPaaSCall?
Name Description

callId

The callId of the call you want to retrieve.

CPaaSCall

A CPaaSCall object that contains all active call-related actions.

endCall

Closes the active call.

func endCall()

getCallId

Returns this call id.

func getCallId() -> String

mute

Mutes yourself, will return a closure with a KotlinBoolean to determine if the operation was successful.

func mute(completion: @escaping (KotlinBoolean) -> Void)
KotlinBoolean is the way kotlin converts its boolean for iOS, you can use it with == and !=.

unMute

UnMutes yourself, will return a closure with a KotlinBoolean to determine if the operation was successful.

func unMute(completion: @escaping (KotlinBoolean) -> Void)
KotlinBoolean is the way kotlin converts its boolean for iOS, you can use it with == and !=.

eventListener

This protocol of type CPaaSCallEvents should be set to receive events on the call.

var eventListener: CPaaSCallEvents? { get set }

Helper Objects

CPaaSCallEvents

The CPaaSCall supports listening to various call events such as: call connected, call ringing, connection failed, call ended and call reconnecting.

onCallEnd

The call ended and you can remove the call UI.

func onCallEnd(reason: CPaaSReason)

onConnected

The call is connected and you are receiving audio.

func onConnected()

onConnectedFailure

The connection failed. You can check the failure reason.

func onConnectedFailure(reason: CPaaSReason)

onReconnecting

The call is in a process of reconnecting and you can show a notification to the user.

func onReconnecting(reason: CPaaSReason)

onRinging

The Callee received your call. You have to wait for them to accept or reject the call.

func onRinging()

CPaaSAPISettings

This class has all parameters needed for registration.

init(customDomain: String, accountSid: String, authToken: String, appSid: String, clientId: String, PNSToken: String, baseURL: String)
Name Description

customDomain

The base domain of your CPaaS organization relative to api.your-company.com

accountSid

A string that uniquely identifies this account.

authToken

The secret authorization token for this account

appSid

The string that uniquely identifies this application.

clientId

The string that uniquely identifies this client.

PNSToken

Push notification server token

baseURL

Base AWS URL. Parameters can be taken from here

CallOptions

This object is used when connecting to a call, you can determine the beginning status of the call with this.

Name Description

audio

You can determine if the call will start with the user muted or not

CPaaSResult

An Object for receiving asynchronous results

onSuccess

If the operation was successful this closure will be invoked with the parameter that was set at declaration.

func onSuccess(action: @escaping (T?) -> Void) -> CPaaSResult<T, TException>

onFailure

If the operation failed this closure will be invoked with the error [type](=cpaas-error) that was set at declaration.

func onFailure(action: @escaping (TException?) -> Void) -> CPaaSResult<T, TException>

fold

With this method you will listen to onSuccess and onFailure closures in same method.

func fold(onSuccess: @escaping (T?) -> Void, onFailure: @escaping (TException?) -> Void) -> KotlinUnit?
Name Description

isSuccess

You can ask CPaaSResult if the operation succeeded

isFailure

You can ask CPaaSResult if the operation failed

CPaaSAPICb

You set this on registration to add a listener for general events.

onIncomingCall

Called when there is an incoming call.

func onIncomingCall(callId: String, callerId: String, serviceType: ServiceType)
Name Description

callId

The id that identifies the call

callerId

The callers id

serviceType

onRegistrationComplete

Indicates about the completion of the registration process.

func onRegistrationComplete(success: Bool)
Name Description

success

Determine if the registration was successful
Even if registration was successful you should continue listening to this event as it may still fail for various reasons.

<a name="cpaasreason"></a>CPaaSReason

The reason why a call ended, failed or is in reconnecting process.

Name Description

busy

Server issue

reject

User reject the call

badRequest

Missing parameters

unauthorized

There is an issue with your token, please check that your account was set correctly and parameters were passed correctly

notFound

Server issue

gone

Server issue

timeOut

Failed to receive response from the server

rtpLost

The rtp was lost, we are trying to reconnect

network

Call ended as result of a network issue

byUser

When the user requested to end the call

byServer

When the callee ended the call

ServiceType

When receiving an incoming call, you can know with this enum the type of the call and which API should be referenced.

Name Description

voice

We are in a voice call, you should use ----CPaaSAPI.shared.voice---- to connect or reject the call

CPaaS Errors

To assist you in getting back on track if you run into problems, we’ve given the following Errors Information. This information can also be helpful for submitting logs and providing details to CPaaS Support.

CallStartError

Errors that can be returned as failure to connect()

ConnectionError

Can’t connect to this callId. Unexpected connection issue prevent the call from being established.

MicrophonePermissionError

Microphone access denied. You should encourage the user to grant Microphone permissions

NotRegisteredError

Call cannot be initiated without registration. can be solved by registering

CallAlreadyExistError

This callId has already been connected. there is already a CPaaSCallobject that represents this call.
