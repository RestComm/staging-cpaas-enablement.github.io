<a name="api"></a>CPaasAPI

CPaaSAPI is your entry point to the CPaaSAPI SDK.

Register

To start using the SDK you should register the SDK by using this function

register(settings: cPaasJsApi.data.JsSettings): Promise<CPaaSEvents>;

logout

This will clear all objects and unregister from receiving further incoming call

logout(): void;

exportLogs

download latest log file

exportLogs(): Promise<void>;

<a name="voice"></a>Voice Object

This interface is your entry point for handling all voice relevant methods.

create

Creates a callId that can be used to start a call or share it with others

create(): Promise<string>;

connect

Call this when you want to start or join a [call](#call). <br/>

Success: Call object will be return <br/> Failure: On Failure will return an exception with an error description.

connect(callId: string, callOptions: cpaas.data.JsCallOptions): Promise<cpaas.data.CPaasCall>;
Name Description

callId

The callId of the call, this id can be retrieved from create, incomingCall or from sharing.

callOptions

This parameter is of type CallOptions to configure some settings when starting the call

getExistingCall

Returns an existing CPaaSCall by callId or null if the call doesn’t exists.

call
getExistingCall(callId: string): Nullable<cpaas.data.CPaasCall>;
Name Description

callId

The callId of the call you want to retrieve.

reject

Use this when you want to reject an incoming call.

reject(callId: string): void;
Name Description

callId

The callId of the call you want to reject

<a name="call"></a>Call Object

A Call object that contains all active call-related actions.

end

Closes the active call.

 end(): void;

mute

Mutes yourself, will return a promise with a Boolean to determine if the operation was successful.

mute(): Promise<boolean>;

unmute

UnMutes yourself, will return a promise with a Boolean to determine if the operation was successful.

unmute(): Promise<boolean>;

Listening to Call Events

The Web SDK supports listening to various call events such as: call connected, call ringing, connection failed, call ended, call reconnecting. You can listen to call events as follows.

onCallEnd

The call ended and you can remove the call UI.

onCallEnd(): cpaas.events.OnCallEnd;

onConnected

The call is connected and you have open channel for sending and receiving audio.

onConnected(): cpaas.events.OnConnected;

onConnectedFailure

The connection failed. You can check the failure reason.

onConnectionFailure(): cpaas.events.OnConnectionFailure;

onReconnecting

The call is in a process of reconnecting and you can show a notification to the user.

onReconnecting(): cpaas.events.OnReconnecting;

onRinging

The Callee received your call. You have to wait for them to accept or reject the call.

onRinging(): cpaas.events.OnRinging;
