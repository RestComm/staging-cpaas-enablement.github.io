Overview

Auto Attendant (AA) is intended for business customers who want to provide an automated means of directing incoming calls to the appropriate person within the organization, typically to offload this task from a receptionist and also offer a simple menu system ("for menu, press 1, for transfer to phone, press 2" etc.)

What is Auto Attendant (AA)?

CPaaS Auto-Attendant (AA) is a voice menu system that allows callers to be transferred to an extension without going through a telephone operator or receptionist.

CPaaS Auto Attendant allows you to create a menu of different options that a caller can access using telephone key presses.The caller hears an announcement listing the options, such as

  • To dial by name press 1.

  • To dial by extension press 2.

  • To hang up press 3.

Caller can then access each of these options by pressing the appropriate key on the telephone keypad. Enterprise Customers can define different menus for an AA system such as out of business hours and on holidays, so that the caller is presented with a different set of options according to the time of the call, or is presented with an announcement that the business is closed and cannot accept calls at this time. You can also include options to allow callers to reach a particular phone extension directly • Dial-by-extension: Callers who know the extension number of the person they want to call can dial this number. The Auto Attendant then connects the caller directly to the requested extension.

How to set up an Auto Attendant?

You must have a valid, working CPaaS account in order to complete this step. If you do not have one, you can contact us.

You can set up your Auto Attendant in two different ways, either via the CPaaS Auto Attendant Console or via the API.

Setting up a new Auto Attendant via Console:

  • Log in to your CPaaS Auto Attendant Console (“https://$DOMAIN/autoattendantconsole/”) via your CPaaS cloud account

  • Create a new Auto Attendant as explained in Auto Attendant system documentation
