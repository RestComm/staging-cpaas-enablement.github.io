Overview

CPaaS Cisco Webex enables you to send and receive SMS using Webex Teams. This particular document explains what happens if a user accidentally removes the bot from the Webex Teams space.

Whenever a new room is created on Webex Teams through CPaaS Cisco Webex, a bot gets added to it. This bot acts as a gateway for sending and receiving messages to and from the Cisco Webex client. If a user deletes the bot from that room, then any new incoming message from that sender will not reach the room.

Blocking the Sender

For an existing room, a user may accidentally remove the bot by clicking the the three dots followed by Remove from space button.

Once the bot is removed from the team the space will look similar to the one below.

As soon as the bot is removed from the space, all the participants in that room will receive a personalized warning message from the bot containing the information about the sender who has been blocked, followed by a link mentioning the steps to follow in order to add the bot back / unblock that sender.

The warning message will appear as a personal message from the bot as shown below.

The message will contain the warning message followed by a link which mentions the steps to be followed in order to unblock that sender.

And if the same sender tries to send a message to the particular mapped number for which the bot has been removed. Then, those messages will start appearing in the BOT space as mentioned below:

The message will contain the warning message along with the message body of the message which was sent by the sender, followed by a link which mentions the steps to be followed in order to unblock that sender.
