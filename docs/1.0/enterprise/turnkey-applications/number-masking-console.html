Number Masking Console

Overview

Number Masking enables Business Customers to easily make communication possible between two or more parties without revealing private information (such as phone number).

Business customers can define, manage and view Applications, Contexts, Masked Number Pool etc.

This document covers the features and functionalities of the Number Masking Console (UI) from Business Customer’s User Point of View.

To find out more about the Number Masking API, please check the API documentation.

To sign in to Console please visit https://<domain-provided-by-csp>/numbermaskingconsole/

Terminology

As part of this documentation, the following terms will be used to describe specific individuals, concepts, and organizations.

Communications Provider

Communications Provider generally refers to any organization providing communications products, applications, capabilities, or services. The term can refer to operators, carriers, providers, suppliers, or any other entity that delivers communications capabilities to business customers.

Business Customer

A business customer that relies on a Communications Provider to deliver communications products, capabilities, or services to customers or end users.

Business Customer

Communications Providers serve Business Customers. Through their own Accounts, Communications Providers can create, manage, or delete Business Customer Sub Accounts. By setting up Sub Accounts for Business Customers, the Communications Provider can easily manage billing and invoices for a specific Business Customer.

Applications

To navigate to the page, click on Applications on the top menu.

Users can view a list of all Applications that have been created for this account/Business Customer.

Applications can be filtered based on the name.

Add Application

To create a new Application:

  • Click on “Add Application” button

  • Enter the name of the application and click the “Done” button.

The name of the application should be unique.

Basic details

To edit an Application click on the name of the Application in the list. Users will view the Basic Details page. On the page, enter the fields and click Save.

The description of the fields to edit an application include:

Default Time to Live: Default time to live for context or session in the application after Contexts last update date or interaction. If not passed, default is 0, which means unlimited time to live.

Application Name: Name of the application.

GeoMatch Level: Where a masked number must be located relative to the Participant’s phone number. For example if the Participant’s Number is “15713316943” and the masked number pool has a phone number starting with Country Code “1”, it will be selected. If not an error will be returned. As of today only “country” is supported. By default it is set to none.

CallBack Url: URL to call when the interaction status changes. To reset an existing value, pass an empty string like “”.

Intercept Callback Url: The URL that the platform will call on each interaction. If the platform receives a 403 status, it will block the interaction, otherwise the interaction will continue.

Out of Context Callback Url: The URL platform shall be called when an inbound call occurs on a closed or non-existent context. If your server responds with valid RCML, the platform will process it. This means that it is possible, for example, to play a message for a call, or redirect a call to another Phone Number.

Masked Numbers

To navigate to this page click on Masked Numbers in the side navigation bar of the screen.

  • Users can view a list of all Masked Numbers in the pool that are assigned to the Application.

  • The Masked Numbers list can be filtered based on the masked number.

  • To mark a Masked Number as reserved/unreserved click on checkbox of a particular masked number.

Add Masked number

To add a masked number click on the “Add Masked number” button.

List of all the Phone Numbers belonging to the Enterprise and not assigned to any Applications are shown. To add a Phone Number as Masked Number, user need to select the checkbox of particular number.

Context

To navigate to this page click on Contexts in the side navigation bar of the screen.

Users can view a list of all Contexts that have been created under the Application.

Contexts can be filtered based on the name of the Context.

Add Context

To add a context click on the Add Context button.

Enter the fields required and click on the “Done” button.

The description of the field to add context:

  • Context Name: Mandatory field and should be unique within Application.

  • Time to Live: Optional field. If provided, it overrides the default time to live configured in the basic details of application .

  • Expiry Date: Optional field,the date when the Context should expire. This will override time to live if configured.

  • Mode: Mode of the interaction, example:Voice.

Edit Context

To edit a Context click on the Context name in the Contexts list page. To go back to the context list page click on the Back button on top right.

Users can view the details of the Context.

  • Edit the Context by updating Name, Expiry Date, Mode, Status, Time to live and click on the “Save Changes” button.

  • Delete the Context by clicking on the “Delete” button

Participants

To view or add the participants in the Context click on Participant link in the Context list page.

To go back to the Context list page click the Back button on the top right.

Users can view the list of Participants which are added under that particular context.

Participants can be filtered based on the Participant Number.

Add Participants

To add a participant in the context click on the “Add Participants” button. Enter the fields required and click on the “Done” button. Description of the fields to add participant:

  • Participant Name: Optional. Name of the participant.

  • Phone number: Mandatory field. Phone number of the participant

  • Masked Number: Optional field. Masked Number available in Masked Number pool of Application.

  • Masked Number SID: Optional field. SID of the Masked Number number.

If both the Masked Number SID and Masked Number are specified, the Masked Number SID takes preference. If both the Masked number and Masked Number SID are not specified, the Application will select a random number from the Masked Number pool.

Participants Details

Click on the Name of the Participant to view the details.

Numbers

Your Communications Provider would have reserved Phone Numbers for your Business to use. Only these numbers can be used as Masked Numbers.

To navigate to the page, click on Numbers located on the top menu.

Users can view a list of all phone numbers that have been assigned for particular Business Customer.

Numbers can be filtered based on the phone numbers.

User Management

To navigate to the page, click on User Management on the top menu.

  • View a list of all users that have been created under an account.

  • Filter based on the User Name.

Add Users

To add Users click on the Add User button in the User management page.

Follow the steps below to add a User:

  • Enter the name of the new User.

  • Enter the email ID of the User.

  • Enter the password for the User Account.

  • Set a role for the user (ADMIN)

  • Set the status of the account (ACTIVE, INACTIVE)

Email ID must be unique. 
Password should meet below criteria:

  • At least one Uppercase letter(A,B,C,D……​)

  • At least one Lowercase letter(a,b,c,d,…​…​.)

  • At least one number(1,2,3,4,5….)

  • At least one special character(!,@,#,$,%,&,…​…​.)

  • Length of the password should be greater than equal to 8.

User Details

Click on the name of the user in list to view the details.

Click on Close Account to delete the user.

Search

Search Application

To navigate to the page, click on Search on the top menu.

User can perform a search on the applications of the Account. Below are the supported filters for application search:

  • Start Date: Shows the list of applications that were created on or after this date.

  • End Date: Shows the list of applications that were created on or before this date.

  • Application Name: Shows the list of applications which name partially or fully matches with the given name.

Application Details

Users can view details of a particular application by clicking any application name in the list.

Search Masked Numbers

To navigate to the page, click on the Masked numbers on the side navigation bar..

Users can perform a search on the Masked numbers of the account. Below are the supported search filters:

  • Start Date: Shows the list of Masked numbers that were created on or after this date.

  • End Date: Shows the list of Masked numbers that were created on or before this date.

  • Application: Shows the list of Masked numbers that belong to the application selected from the dropdown.

  • Is Reserved: Shows the list of Masked numbers with the Is Reserved status (True, False).

  • In Use: Shows the list of Masked numbers with the In Use status (True, False).

  • Masked number: Only show Masked Number that starts with phoneNumber digits. This is useful to search Masked Numbers that belong to specific country code.

Masked Number Details

Users can view details of a particular Masked Number by clicking any SID in the list.

Search Context

To navigate to the page, click on Contexts on the side navigation bar.

Users can perform a search on the Contexts of the account. Below are the supported filters for Context search:

  • Start Date: Shows the list of Contexts that were created on or after this date.

  • End Date: Shows the list of Contexts that were created on or before this date. A* pplication: Shows the list of Contexts that belong to the application selected from the dropdown.

  • Status: Shows the list of Contexts that matches with this status (‘Open’, ‘In progress’ , ‘Closed’)

  • Context Name: Shows the list of Contexts with a name that partially or fully matches with the given name:

Context Details

Users can view details of a particular Context by clicking any Context name in the list.

Search Call Logs

To navigate to the page, click on the Call Logs on the side navigation bar. A User can perform a search on the Call Logs of the Account. Below are the supported filters for Call Logs search:

  • Account Scope:

    • Search in Current Account: Shows the list of call logs that are created by the logged in user.

    • Search in current Account and Sub Accounts: Shows the list of all the call logs that are created by the logged in user and their Sub Accounts.

    • Search in Specific Sub Account: To search call logs that are created by a particular Sub Account User. The User is presented with a list of Sub Account Users in a popup window. The User can select the Sub Account on that popup window.

  • Start Date: Shows the list of call logs that were created on or after this date.

  • End Date: Shows the list of call logs that were created on or before this date.

  • Application: Shows the list of call logs that belong to the application selected from the dropdown.

  • Context SID: Shows the list of call logs with a Context SID that partially or fully matches the entered Context SID.

Usage Records

The Usage Record REST resource provides a simple API to retrieve usage records of Number Masking application during any period of time and in any mode (Voice, Message or VoiceAndMessage). This makes it easy to build reporting and analytics tools for your application.

Usage Records for Number Masking consist of Contexts created, Voice minutes including number of calls, and SMS Units.

Usage Records data is collected every 15 minutes, so there is possibility that once Context/Interaction is completed, it may take a maximum of 15 minutes to be reflected in the Usage Records.

Context

To navigate to Context click on Usage Records → Context.

Below is a list of all supported filters you can use to search Context Usage Records.

  • Subresource: The main usage records list resource supports a variety of convenient subresources like Daily, Monthly etc. If you would like to see consolidated usage records for specific time period, avoid using this filter.

  • Start Date: List Usage Records starting from the specified start date filter.

  • End Date: List Usage Records ending at the specified end date filter.

  • Application: List Usage Records for the specified Application filter.

  • Category: List Usage Records for the specified Context mode.
