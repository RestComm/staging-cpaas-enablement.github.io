Account Roles

Overview

CSPs can create accounts using Restcomm that can later bet used for segmenting users phone numbers and usage statistics and controlling access to data.

Account Roles

Each newly created account can have one of the following roles assigned: Administrator, Developer or ProvisioningAgent.

Administrator

Administrator has full access rights on all platform resources. He can create multiple sub-accounts under himself and can access and manage their resources.

Developer

Developer is very similar to Administrator but has no right to manage sub-accounts and read the organization’s data. In addition, Developer cannot access the OutgoingCallerId API.

ProvisioningAgent

ProvisioningAgent cannot create and manage sub-accounts and has no rights to read the actual text of SMS messages or delete Recordings.

The role of a sub-account can be changed at any time by the user who created it.
