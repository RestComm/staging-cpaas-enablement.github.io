CPaaS iOS SDK Integration Guide

Overview

The CPaaS API iOS SDK enables you to initiate and receive calls, and listen to call events. This guide will walk you through the process of downloading the CPaaS iOS SDK, importing it into your project and testing its capabilities.

Prerequisites

Before you get started, make sure you have the following prerequisites:
1. Xcode v.12.0+
2. iOS v.13.0+

Downloading the CPaaS iOS SDK

The CPaaS API iOS SDK is available for download here.

Manual Install

You can install the SDK manually by following the steps below.

  1. Download CPaaSAPI.framework and WebRTC.framework and unpack them in your project.

  2. Drag and drop CPaaSAPI.framework and WebRTC.framework onto the Frameworks, Libraries, and Embedded Content section that is located in Xcode General tab.

  3. Ensure that "Copy items if needed" is checked and press Finish.

  4. Ensure that "Embed & Sign" is selected.

Initialize the SDK

Before you start using the SDK you will need to initialize it first. To do so, you will have to register the application by running the script below.

let settings = CPaaSAPISettings(
            customDomain: "domain",
            accountSid: "sid",
            accountToken: "token",
            appSid: "sid12345",
            clientId: "foobar22",
            PNSToken: "pns55555",
            BaseURL: "https://cpaas.dev:3000")

CPaaSAPI.shared.register(settings: settings, cpaasAPICb: self)

extension CLASS_NAME: CPaaSAPICb {

    func onIncomingCall(call: ICall) {
        //TODO: - Listening to incoming calls
    }

    func onRegistrationComplete(success: Bool) {
        //TODO: - You can start your call from here
    }
}

Authorization and Web Socket Connection

If the registration of the application is successful, the following two events will happen.

  • Authorization - Our servers will authenticate and authorize the use of the SDK.

  • Web socket connection initialization - We will initialize the web socket connection needed by the SDK.

The previous steps must be successfully completed. Failure in any of the above procedures will prevent the SDK from working properly.

To start using the SDK you will need to invoke the .register() method by running the following script. This can only be done once. A second invocation will take no effect on the SDK.

CPaaSAPI.shared.register(settings: settings, cpaasAPICb: self)
CPaaSAPI.shared.register(settings: settings, cpaasAPICb: self) //A second invocation will have no effect

Starting a call

To initiate a call simply call CPaaSAPI.shared.startCall(…​). <br>

let call = CPaaSAPI.shared.startCall(
            destinationId: destinationID,
            callOptions: CallOptions(audio: true))

call is an API object that lets you perform actions on the active call such as muting or putting it on hold.

call.mute() (1)
call.hold() (2)
1 Mutes the current call
2 Puts the current call on hold

Answering an Incoming Call

To answer an incoming call, you need to listen to the onIncomingCall callback.

When there is an incoming call, you can use the call object to accept it and call call.join() to join it.

func onIncomingCall(call: ICall) {
    //Save the call object
    call.joinCall()
}

When working with callKit you will want to first reportNewIncomingCall and then call joinCall() or rejectCall() depending on the CXCallAction.

Listening to call events

You can listen to various call events such as: call connected, call ringing, connection failed, call ended, call reconnecting.

call.eventListener = self

extension CLASS_NAME: ICallEvents {

    func onRinging() {
        // callee got your call, wait for his action (accept/reject)
    }

    func onConnected() {
        // call connected you have audio here
    }

    func onCallEnd(reason: Reason?) {
        // call end, you can remove call UI
    }

    func onConnectedFailure(reason: Reason) {
        // connection failed, check reason
    }

    func onReconnecting(reason: Reason) {
        // call on reconnecting process, show notification for user
    }
}

API Limitations

Be aware that all API methods must be called on the main thread.
