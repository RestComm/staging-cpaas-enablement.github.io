Android SDK API Reference

Main Objects

The Android SDK is using the following main object:

Property Description

CPaaSAPI

Main API object.

startCall

An object used to call a destination.

Call

A call object that contains all call-related operations.

CPaaSAPISettings

Settings required when making requests to the API.

The CPaaSApi Object

Functions

Property Description Returns

register

Use this method to initialize the API. Can be called once per page load.

Use this method to start a call.

ICall

logout

Use this method to logout.

Void

Use this method to update credentials.

Void

Returns an existing Call by callId or null if the call doesn’t exists.

ICall

Returns local file log provider object that holds the log file.

The CPaaSAPISettings object (Settings)

Property Description

customDomain

Your custom Domain

accountSid

The Restcomm identifier of the Account where the event happened

accountToken

The authorization token for this account. This should not be shared

appSid

The application unique identifier for a specific account

clientId

Client unique identifier. used for authentication (by Cognito)

BaseURL

Your Base URL

WS URL

Events

Registration Events

Name Description

onIncomingCall

A call object that contains all call-related operations.

onRegistrationComplete

Indicates about the completion of the registration process.

Call Events

Name Description

onConnected

The call is connected and you are receiving audio.

onRinging

The Callee received your call. You have to wait for them to accept or reject the call.

onConnectedFailure

The connection failed. You can check the failure reason.

onCallEnd

The call ended and you can remove the call UI.

onReconnecting

The call is in a process of reconnecting and you can show a notification to the user.

Connected Failure Reasons

Name Description

reason

The reason why a call failed. Possible values include: <br/> BUSY, REJECT, BAD_REQUEST, UNAUTHORIZED, NOT_FOUND, GONE,RTP_LOST
