Call SDR

Call records are generated for incoming or outgoing voice traffic.

Format

A Call SDR record comprises the following fields.

Field Description Type

api_version

The API version that generated the record.

string

sid

A string that uniquely identifies this call.

string

instance_id

The unique id of the Instance that handled this call.

string

account_sid

The unique id of the Account that created this call.

string

account_friendly_name

The friendly name of the Account.

string

org_sid

The unique id of the Organization which belongs to the Account.

string

org_name

The domain name of the Organization which belongs to the Account.

string

parent_call_sid

The unique id of the parent Call that created this call.

string

conference_sid

The unique id of the Conference created by this call.

string

date_created

The date on which this call was created.

timestamp (format: Mon, 04 Jul 2022 01:46:26 +0000)

date_created_ms

The datetime in milliseconds on which this call was created.

timestamp (format: Mon, 04 Jul 2022 01:46:26.123 +0000)

date_updated

The date on which this call was last updated.

timestamp (format: Thu, 01 Jan 2022 00:20:00 +0000)

date_updated_ms

The datetime in milliseconds on which this call was last updated.

timestamp (format: Thu, 01 Jan 2022 00:20:00.123 +0000)

direction

A string describing the direction of the call.

Possible values are inbound, outbound-api, and outbound-dial.

string

phone_number_sid

If the call was inbound, this is the Sid of the IncomingPhoneNumber that received the call.

string

from

The phone number or identifier that originated this call.

string

to

The phone number or identifier that is the recipient of this call.

string

caller_name

If this call was an incoming call, the caller’s name. Empty otherwise.

string

forwarded_from

If this call was an incoming call forwarded from another number, the forwarding phone number (depends on if the carrier supports forwarding). Empty otherwise.

string

answered_by

If this call was initiated with answering machine detection, either human or machine. Empty otherwise.

string

start_time

The start time of the call. Empty if the call has not yet started.

timestamp (format: 2022-01-01T00:00:00.000Z)

start_time_ms

The start datetime milliseconds of the call. Empty if the call has not yet started.

timestamp (format: Thu, 01 Jan 2022 00:20:00.123 +0000)

end_time

The end time of the call. Empty if the call has not ended yet.

timestamp (format: 2022-01-01T00:00:00.000Z)

end_time_ms

The end time of the call in milliseconds. Empty if the call has not ended yet.

timestamp (format: Thu, 01 Jan 2022 00:20:00.123 +0000)

ring_duration

The ringing duration of the call in seconds.

integer

duration

The length of the call in seconds.

integer

price

The cost of the call. It always defaults to 0.0.

double

price_unit

The price per unit.

string

status

A string representing the status of the call.

Possible values are: queued, ringing, in-progress, completed, failed, busy and no-answer.

string

sip_status_code

The status code of the last SIP response.

integer

sip_status_reason

The status message of the last SIP response.

string

custom_headers.*

Any custom header associated with this call. May result in multiple columns or none.

string

reason.message

The message explaining the call failure.

string

reason.code

The reason code explaining the call failure. Relative to reason.protocol.

integer

reason.protocol

The protocol that originated the call failure. Relative to reason.code.

string

uri

The URI that references the SDR record.

URI

subresource_uris.notifications

The location of Notification related to this call.

URI

subresource_uris.recordings

The location of Recording related to this call.

URI
