Tags

Restcomm’s Account API supports billing tags to be passed in the core network messages, in the various protocols and be persisted in the SDRs for easier billing reconciliation.

An Enterprise Account can be assigned a maximum of 5 tags.Each tag name must be unique.

Create Tag

POST /restcomm/2012-04-24/Accounts/{AccountSid}/Tags.json

Creates a Tag, assigns it to accountSid and returns its representation.

Tags can only be created by the CSP Administrator.
Body Parameter Description

Name

Tag name (Required).

Value

Tag value (Required).

Type

Tag type (Required). The only possible value is Billing.

Examples

Create a Billing tag named msisdn with value +30XXXXXXXXX and assign it to an accountSid:

curl -X POST https://mycompany.restcomm.com/restcomm/2012-04-24/Accounts/ACCOUNT_SID/Tags.json  \
   -d 'Name=msisdn' \
   -d 'Value=+30XXXXXXXXX' \
   -d 'Type=Billing' \
   -u 'YourAccountSid:YourAuthToken'

const request = require('request');

// Provide your Account Sid and Auth Token from your Console Account page

Update Tag

POST /restcomm/2012-04-24/Accounts/{AccountSid}/Tags/{TagSid}.json

Updates a Tag assigned to accountSid and returns its representation.

Tags can only be updated by the CSP Administrator.
Body Parameter Description

Name

Tag name (Optional).

Value

Tag value (Optional).

Type

Tag type, so far the only possible value is Billing (Optional).

Examples

Update Tag with Sid TG9e21ed2cb5e548a6bcafbae1e14c238e and set Value to 123456789:

curl -X PUT https://mycompany.restcomm.com/restcomm/2012-04-24/Accounts/ACCOUNT_SID/Tags/TAG_SID.json  \
   -d 'Value=123456789' \

Get Tags List

GET /restcomm/2012-04-24/Accounts/{AccountSid}/Tags.json

Return a list of Tags belonging to an `accountSid.

Tags can only be queried by the CSP Administrator.

Filtering

The following filtering parameters are supported.

Query Parameter Description

Type

Only return Tags of specified Type. Currently only type Billing is supported.

Examples

Query all tags assigned to accountSid of Type Billing.

curl -X GET https://mycompany.restcomm.com/restcomm/2012-04-24/Accounts/ACCOUNT_SID/Tags.json?Type=Billing  \

Return detailed information for a single Tag identified by the TagSid path parameter in the URL.

Tags can only be queried by CSP Administrators

Tags Attributes

The following is a list of attributes of the Tags resource:

Attribute Description

Sid

A string that uniquely identifies the Tag.

Name

The name of the tag.

Value

The value of the tag.

Type

The type of the tag (currently supporting type Billing).

DateCreated

The date on which this Tag was created.

DateUpdated

The date on which this Tag was last updated.

OwnerSid

The AccountSid for the owner of this Tag.

TargetSid

The AccountSid where the Tag is assigned.

Uri

The suffix for the HTTP resource, relative to the base organization url. For example when creating a Tag, the Uri will be /2012-04-24/Accounts/ACXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX/Tags/TGf0a2ca58c89b41bc829a319371a7a959.json.

Passing Custom Tags in SMPP TLV Mapping

You can map billing Tags or Restcomm core network headers to SMPP TLVs for billing reconciliation or other purposes.

PUT /restcomm/2012-04-24/Profiles/{ProfileSid}`

Subscriber Profiles can only be modified by the CSP Administrators that.

Example Request

{
    "multiProvider": {
        "outbound-sms": [
            {
                "name": "your-network-profile",
                "serverIpAddress": "10.0.0.1:9999",
                "version": "3.4",
                "systemId": "esmeid",
                "password": "esmepass",
                "sessionType": "CLIENT",
                "headerTranslationMap": {
                    "msisdn": "X-TLV-COCTET-5",
                    "secondtagname": "X-TLV-1BYTE-3"
                }
            }
        ]
    }
}

Request Parameters

Attribute Description

name

The name of your profile

serverIpAddress

The IP address of the server

version

The SMPP version you are using.

systemId

The SystemID and port used in BIND request to SMSC. E.g. 10.0.0.1:9999

password

The password used in BIND request to SMSC.

sessionType

Either a CLIENT or SERVER. Specifies how Restcomm will behave when interconnecting with your SMPP infrastructure. Restcomm will only allow Transceiver sessions.

headerTranslationMap

Your TLV mappings list. A single mapping has the following format: "tag_name": "X-TLV-<type(1BYTE|2BYTE|4BYTE|OCTET|COCTET)>-<id>". The TLV ID is a decimal that is converted to hexadecimal when mapped to SMPP.
For more information about SMPP specification, please visit https://smpp.org/.

Distributed Request Processing

The Restcomm platform has a distributed microservice architecture, that enables linear scalability during traffic growth and handles a high volume of REST API requests, voice calls, and SMS messages. Each architecture component handles different types of requests.

When you modify the provisioning data via the Management REST APIs, up to few minutes are needed for the update to reflect the changes in all call processing units that handle calls and SMS. Provisioning data modifications are done using the HTTP POST, PUT and DELETE methods. The POST method updates records immediately with no delay during the operation.

Distributed request processing is available for the following Management APIs, as well as the Restcomm Console.

  • Accounts

  • Tags

  • Profiles

  • Clients

  • Incoming Phone Numbers
