Accessing custom SIP headers from an RCML or Visual Designer application

Restcomm provides an additional interface to communicate with an RCML/Visual Designer application by means of custom SIP headers. The idea is that all custom SIP headers in a SIP message are detected and converted to HTTP parameters in the request to the RCML application using a convention. The RCML application can consume those parameters. Furthermore, Visual Designer converts them to RVD variables and makes them available to its modules.

RCML Applications

The following conventions are followed:

  • Any SIP header starting with X- prefix is custom.

  • HTTP parameters are created out of each custom SIP header by adding the SipHeader_ prefix to its name.

Mapping

Below is an example of mapping from SIP header name to HTTP parameter name:

X-My-Header -> SipHeader_X-My-Header

Thus the SipHeader_X-My-Header will be available as typical GET or POST parameter in the RCML application.

Visual Designer Applications

Visual Designer will take the previous convention one step further and create a Visual Designer variable out of the parameter. Thus, the mapping from a custom SIP header to an Visual Designer variable will be the following:

X-My-Header -> $core_x_my_header

If X-My-Header SIP header is received by Restcomm, Visual Designer will have a variable named $core_x_my_header available in its modules.

  • All characters are converted to lower case. This happens because although SIP headers are not case insesitive while Visual Designer variables are.

  • All non alpha numeric characters i.e.[^A-Za-z0-9_], are converted to `_' (underscore). That happens because Visual Designer has some additional restrictions for naming variables. Here is the regular expression that is used for invalid characters to underscore:

Passing Custom Headers Using Dial Verb

When editing a voice application in Visual Designer you can use the Dial → SIP URI and provide a header there to be used in the generated call. For example, using: sip:jane@acme.restcomm.com?X-Custom-Header=my_value will result in the following RCML:

<Response>
	 <Dial>
		  <Sip>sip:jane.doe@acme.restcomm.com?X-Custom-Header=my_value</Sip>
	 </Dial>
</Response>

Passing Custom Headers Using WebTrigger

You can pass parameters in the WebTrigger query string which Restcomm will then propagate and make module parameters in Visual Designer. For example, using https://frontend-tests-org-staging.restcomm.com/visual-designer/services/apps/AP184cf3eb79a9496f926c9e3bd15988ae/start?from=12345&to=client:jane&testqueryparam=0value0 will make Restcomm send module_module_testqueryparam=0value0 in its RCML request (as query string if the GET method is used and form-urlencoded if the POST method is used), which will be then available as $testqueryparam in the Visual Designer application.
